# Data Merge Utility (DMU) 🚀

**Data Merge Utility** 是一款基于 Python 和 PySide6 开发的高效、美观的工业级数据处理工具。它通过"模板学习"与"多文件合并"两阶段工作流，解决了多源数据合并过程中表头对齐、清洗策略不统一及标准化导出的痛点。

---

## ✨ 核心特性

* **🧠 模板学习 (Template Learning)**
    * **自动特征探查**：导入样表即可自动识别数据类型、空值占比及唯一值分布。
    * **规则可视化**：直接在表格中为每一列配置"空值处理"及"重复值策略"。
    * **配置持久化**：将策略一键导出为标准 JSON 模板，实现业务逻辑与程序的分离。

* **🔗 智能多文件合并 (Multi-File Merge)**
    * **动态模板加载**：内置实时扫描机制，支持在软件不重启的情况下切换不同的 JSON 配置文件。
    * **高兼容性逻辑**：支持"子集匹配"，只要文件包含目标列即可合并，多余列自动剔除。
    * **异步处理流**：基于 `QThread` 异步执行，配合实时进度条与日志监控，处理大批量数据不卡死。

* **💾 专业化导出**
    * **格式自适应**：支持合并后数据的实时 Top 5 预览。
    * **报表级样式**：一键导出包含"等线"字体优化、自动调整列宽及无框线美化的标准 .xlsx 文件。

* **🎨 工业风视觉 (Morandi Style)**
    * 深碳灰与莫兰迪蓝的高级配色，1600x800 分辨率适配，极简且专业。

### 各模块职责说明

#### 1. MainApp.py - 主程序入口

**功能**：作为应用程序的"大脑"，负责整体窗口管理和页面导航。

**具体工作**：
- 创建主窗口 (1840x800 尺寸)
- 构建左侧导航栏（侧边栏）
- 管理右侧内容区域的页面切换
- 维护统一的莫兰迪工业风格样式

**与其它模块的关系**：
- 创建 `TemplateLearningPage` 和 `MultiFileMergePage` 的实例
- 当用户点击导航按钮时，切换显示不同的页面
- 不直接处理数据，只负责"调度"工作

**简单比喻**：就像一个公司的前台接待员，负责引导来访者到正确的部门。

---

#### 2. TemplateLearningPage.py - 模板学习页面（模块A）

**功能**：帮助用户定义数据清洗规则，并将这些规则保存为配置文件。

**具体工作**：
1. **导入数据**：允许用户选择 Excel 或 CSV 文件
2. **自动分析**：调用 DataProfilingEngine 分析数据特征
3. **可视化配置**：显示每个字段的数据类型、空值比例等信息
4. **规则设置**：用户可以为每列设置"空值处理策略"和"重复值策略"
5. **导出配置**：将所有设置保存为 `模板.json` 文件

**与其它模块的关系**：
- 依赖 `DataProfilingEngine` 进行数据分析
- 将分析结果和用户配置输出到 `模板.json`
- 为 `MultiFileMergePage` 提供配置文件

**核心流程**：
```
用户导入文件 → ProfilingWorker线程 → DataProfilingEngine分析 → 
界面显示分析结果 → 用户设置清洗策略 → 保存为模板.json
```

**使用的关键技术**：
- `QThread` 异步处理，避免界面卡顿
- 自定义信号 (`Signal`) 进行线程通信

---

#### 3. DataProfilingEngine.py - 数据探查引擎

**功能**：底层数据分析引擎，负责"认识"数据的特征。

**具体工作**：
1. **数据类型识别**：自动判断每列是数字、日期、文本、布尔值还是分类数据
2. **统计信息计算**：计算空值比例、唯一值数量等
3. **异常检测**：识别数值列中的异常值（基于3σ原则）
4. **建议生成**：为文本列提供模糊匹配建议（如发现相似但不完全相同的值）

**与其它模块的关系**：
- 被 `TemplateLearningPage` 调用
- 接收 Pandas DataFrame，返回详细的分析报告
- 不直接与 GUI 交互，只做纯数据处理

**数据类型识别顺序**（瀑布流方式）：
```
Boolean → Numeric → DateTime → Categorical → String
```

**简单比喻**：就像一个"数据医生"，给数据做全面体检并出具报告。

---

#### 4. MultiFileMergePage.py - 多文件合并页面（模块B）

**功能**：利用模板配置，对多个文件进行合并、清洗和导出。

**具体工作**：
1. **模板加载**：扫描并加载用户选择的 JSON 配置文件
2. **文件选择**：允许用户添加多个待合并的文件
3. **异步合并**：在后台线程中执行合并操作
4. **应用清洗策略**：根据模板配置处理空值和重复值
5. **数据预览**：显示合并结果的前5行
6. **导出美化**：导出为格式优美的 Excel 文件

**与其它模块的关系**：
- 读取 `模板.json` 获取清洗规则
- 调用 `MergeWorker` 线程执行合并
- 使用 `openpyxl` 进行 Excel 样式美化

**核心流程**：
```
用户选择模板 → 用户添加文件 → 点击执行合并 → 
MergeWorker线程处理 → 按模板清洗数据 → 用户预览 → 导出Excel
```

**使用的关键技术**：
- `QThread` 异步处理长时间运行的任务
- 信号机制实时更新进度条和日志
- `openpyxl` 控制导出样式

---

#### 5. 模板.json - 配置文件（桥梁角色）

**功能**：存储数据清洗规则的配置文件，连接"模板学习"和"多文件合并"两个阶段。

**内容结构**：
```json
{
    "headers": ["列名1", "列名2", "列名3"],
    "column_settings": {
        "列名1": {
            "null_policy": "保留空值/删除空值行/填充默认值",
            "duplicate_policy": "保留重复/仅留首行"
        },
        "列名2": { ... }
    }
}
```

**与其它模块的关系**：
- 由 `TemplateLearningPage` 生成
- 被 `MultiFileMergePage` 读取和使用
- 实现"业务逻辑"与"程序代码"的分离

**简单比喻**：就像一份"工作手册"，告诉后续步骤该如何处理数据。

## 🔄 模块协作示例

### 场景：用户要将多个销售报表合并为一个统一文件

**步骤一：模板学习阶段**
```
1. 用户打开程序，进入"模板学习"页面
2. 导入一个标准格式的Excel文件作为参考
3. DataProfilingEngine 自动分析各列的数据类型和特征
4. 用户为每列设置清洗规则（如"订单号"列删除重复、"金额"列填充默认值）
5. 点击"保存配置"，生成 模板.json
```

**步骤二：多文件合并阶段**
```
1. 用户切换到"多文件合并"页面
2. 选择刚才生成的 模板.json
3. 添加多个待合并的销售报表
4. 点击"执行合并"
5. MergeWorker 读取模板配置，按规则清洗并合并所有文件
6. 用户预览结果，确认无误后导出
```

**整个流程中数据流向**：
```
Excel/CSV文件 → TemplateLearningPage → DataProfilingEngine 
    → 用户配置 → 模板.json ← MultiFileMergePage 
    → MergeWorker处理 → 合并后数据 → 导出Excel
```

## 🚀 快速开始

### 1. 环境准备

确保已安装 Python 3.10 以上版本，通过 pip 安装必要的依赖库：

```bash
pip install -r requirements.txt
```

或者单独安装：

```bash
pip install pyside6 pandas openpyxl numpy scipy holidays rapidfuzz python-dateutil
```

### 2. 运行程序

在终端或命令行执行主入口文件：

```bash
python MainApp.py
```

### 3. 使用流程

1. **模板学习**：导入一个标准格式的文件，设置每列的清洗规则，保存为 `模板.json`
2. **多文件合并**：选择刚才保存的模板，添加多个待处理文件，执行合并
3. **导出**：预览合并结果，导出为美化的 Excel 文件

---

## 📝 注意事项

- 所有耗时操作都采用异步处理，不会阻塞界面
- 程序会自动保持原始数据的列顺序
- 导出的 Excel 文件使用"等线"字体，无边框设计，适合直接打印或展示
